Access checklist

Private credentials, public controls

Drive credentials remain in backend environment variables and are never stored in browser configuration. A folder may be owned by a different Google account than Firebase; the active Drive identity only needs permission to access it.

Service account: enable Drive API in that service account's Cloud project and use a Shared Drive for uploads, or explicitly share supported content with it.
Google-user OAuth: authorize the human account that owns the My Drive folder using a client secret and refresh token.
Never use a Drive API key for private-folder authentication. Save the folder ID, then run the connection test.

/media command

Random Drive media delivery

Images + MP4

/media selects any supported file. /media type:image or /media type:video filters the Drive folder. It only works inside the configured media channel.

Supported images: .png, .webp, .jpg, .jpeg. Supported videos: .mp4. Private files are attached directly when they fit the limit. Larger files are securely streamed through the bot backend so Discord can still show the image or native video player.
